# Crypto 5/15/30 统一窗口因子方案

最后更新：2026-04-30。

## 1. 目的

当前 32 维外生因子里混用了 `5`、`14`、`20`、`30`、`60` 等窗口，解释成本较高，也不利于后续做消融实验。本方案先不改代码，只总结下一版准备使用的因子结构。

核心原则：

- 所有带窗口的技术指标只使用 `5 / 15 / 30`。
- 基础 K 线仍建议使用 `1min`，所以 `5 / 15 / 30` 分别代表过去约 5 分钟、15 分钟、30 分钟。
- 不使用 K 线形态因子，例如上影线、下影线、实体比例、振幅。
- 保留趋势类指标中的 Bollinger 和 MACD，但 MACD 使用由 `5 / 15 / 30` 组成的窗口对，不再使用传统 `12/26/9`。
- 现货和永续共用大部分技术指标；永续额外加入 funding、OI、basis 等合约结构因子。

## 2. 为什么统一成 5 / 15 / 30

当前主要预测目标是 h30，即预测未来约 30 根 1 分钟 K 线后的收益。输入窗口选择 `5 / 15 / 30` 的含义如下：

| 窗口 | 对 1min K 线的含义 | 作用 |
| --- | --- | --- |
| `5` | 过去约 5 分钟 | 捕捉极短线冲击、快速反转、短动量 |
| `15` | 过去约 15 分钟 | 捕捉中短线动量，降低单根噪声 |
| `30` | 过去约 30 分钟 | 表示接近 h30 预测目标尺度的当前状态 |

使用 `30` 不会造成未来信息泄漏，因为它只使用 `t` 及以前的过去 30 根 K 线。它和 h30 label 的区别是：

```text
输入因子 ret_30：过去 30 分钟收益
训练 label h30：未来 30 分钟收益
```

## 3. 现货核心因子

现货第一版建议使用 21 个因子：

```text
ret_5
ret_15
ret_30

rsi_5
rsi_15
rsi_30

vol_std_5
vol_std_15
vol_std_30

ma5_dev
ma15_dev
ma30_dev

boll_z_5
boll_z_15
boll_z_30

macd_5_15
macd_15_30

market_ret_30
market_vol_30
hour_sin
hour_cos
```

### 3.1 收益 / 动量：`ret_5`、`ret_15`、`ret_30`

计算方式：

```text
ret_N = log(close_t / close_{t-N})
```

含义：

- `ret_5`：过去 5 分钟累计涨跌。
- `ret_15`：过去 15 分钟累计涨跌。
- `ret_30`：过去 30 分钟累计涨跌。

用途：

- 判断市场最近是上涨、下跌还是横盘。
- `ret_5` 更敏感，噪声更大。
- `ret_30` 更稳定，更接近 h30 预测目标的时间尺度。

不再单独使用 `roc_5 / roc_15 / roc_30`，因为 ROC 和 log return 表达的信息高度重复。

### 3.2 涨跌强弱：`rsi_5`、`rsi_15`、`rsi_30`

计算方式：

```text
delta_t = close_t - close_{t-1}
gain_N = rolling_mean(max(delta, 0), N)
loss_N = rolling_mean(max(-delta, 0), N)
RSI_N = 100 - 100 / (1 + gain_N / loss_N)
rsi_N = RSI_N / 100 - 0.5
```

含义：

- `rsi_5`：最近 5 分钟买卖强弱，反应最快。
- `rsi_15`：中短线买卖强弱。
- `rsi_30`：接近 h30 尺度的强弱状态。

用途：

- 高 RSI 表示近期上涨 K 线占优，多头更强。
- 低 RSI 表示近期下跌 K 线占优，空头更强。
- 不再使用传统 `rsi_14`，因为它不符合统一窗口原则。

### 3.3 波动率：`vol_std_5`、`vol_std_15`、`vol_std_30`

计算方式：

```text
one_bar_ret_t = log(close_t / close_{t-1})
vol_std_N = rolling_std(one_bar_ret, N)
```

含义：

- `vol_std_5`：过去 5 分钟收益波动。
- `vol_std_15`：过去 15 分钟收益波动。
- `vol_std_30`：过去 30 分钟收益波动。

用途：

- 判断当前市场是安静还是剧烈波动。
- 波动率高时，价格更容易出现大幅变化，但方向不一定明确。

不再使用 `atr_14` 和 `parkinson_20` 作为第一版核心因子，因为它们引入了不统一窗口。

### 3.4 均线偏离：`ma5_dev`、`ma15_dev`、`ma30_dev`

计算方式：

```text
MA_N = rolling_mean(close, N)
maN_dev = close_t / MA_N - 1
```

含义：

- 当前价格相对 N 分钟均价的偏离。
- 正值表示当前价格在均线上方。
- 负值表示当前价格在均线下方。

用途：

- 判断短线趋势强弱。
- `ma5_dev` 更接近瞬时偏离。
- `ma30_dev` 更接近局部趋势状态。

### 3.5 布林位置：`boll_z_5`、`boll_z_15`、`boll_z_30`

计算方式：

```text
MA_N = rolling_mean(close, N)
STD_N = rolling_std(close, N)
boll_z_N = (close_t - MA_N) / (2 * STD_N)
```

含义：

- 价格处在过去 N 分钟均值附近时，`boll_z_N` 接近 0。
- 价格接近上轨时，`boll_z_N` 偏正。
- 价格接近下轨时，`boll_z_N` 偏负。

用途：

- 判断价格相对近期波动区间的位置。
- 相比 `maN_dev`，Bollinger 会考虑近期波动大小。

为什么保留：

- 它和均线偏离不同，Bollinger 是“带波动率归一化的趋势/位置指标”。

### 3.6 MACD 趋势动能：`macd_5_15`、`macd_15_30`

传统 MACD 使用 `12/26/9`，这不符合统一窗口原则。本方案改为只使用 `5 / 15 / 30` 组成的窗口对。

计算方式：

```text
macd_5_15 = EMA(close, 5) - EMA(close, 15)
macd_15_30 = EMA(close, 15) - EMA(close, 30)
```

建议再做尺度处理：

```text
macd_5_15 = macd_5_15 / close_t
macd_15_30 = macd_15_30 / close_t
```

含义：

- `macd_5_15`：极短线趋势动能。
- `macd_15_30`：中短线趋势动能。

用途：

- 当短 EMA 高于长 EMA，说明短线价格强于较长窗口均值。
- 当该值扩大，说明趋势动能增强。

为什么不是 `macd_5 / macd_15 / macd_30`：

- MACD 天然需要 fast/slow 两个窗口。
- 用 `5/15` 和 `15/30` 两个窗口对，既保持统一窗口，又避免额外引入 `3`、`10`、`12`、`26` 等窗口。

### 3.7 市场环境：`market_ret_30`、`market_vol_30`

建议使用 BTC/USDT 作为 reference market。

计算方式：

```text
market_ret_30 = log(btc_close_t / btc_close_{t-30})
market_vol_30 = rolling_std(log(btc_close_t / btc_close_{t-1}), 30)
```

含义：

- `market_ret_30`：过去 30 分钟市场整体方向。
- `market_vol_30`：过去 30 分钟市场整体波动状态。

用途：

- 帮助模型区分“单币自身信号”和“整个市场都在涨跌”。
- 对 altcoin 特别重要，因为很多币会受 BTC 市场 beta 影响。

### 3.8 时间周期：`hour_sin`、`hour_cos`

计算方式：

```text
hour = 当前 UTC 小时 + minute / 60
hour_sin = sin(2π * hour / 24)
hour_cos = cos(2π * hour / 24)
```

含义：

- 表示一天 24 小时内的位置。
- 使用 sin/cos 是为了让 23:59 和 00:00 在数值上相邻。

用途：

- Crypto 是 24/7 交易，但不同 UTC 时段流动性不同。
- 欧美时段、亚洲时段的波动和成交活跃度可能不同。

## 4. 永续合约额外因子

永续在现货 21 个因子基础上，额外加入 7 个合约结构因子：

```text
funding_rate_z

oi_change_5
oi_change_15
oi_change_30

basis_z_5
basis_z_15
basis_z_30
```

永续合约总计 28 个因子。若保持模型 `n_exog=32` 不变，剩余 4 维填 0 pad。

### 4.1 资金费率异常：`funding_rate_z`

计算方式：

```text
funding_rate_z = rolling_z(funding_rate)
```

含义：

- 资金费率为正，通常说明多头更拥挤。
- 资金费率为负，通常说明空头更拥挤。
- 做 z-score 后，表达的是“当前 funding 相对自身历史是否异常”。

为什么不做 `funding_rate_5/15/30`：

- funding 通常不是每分钟更新，而是按交易所规则周期性结算。
- 强行套 `5/15/30` 没有明确交易含义。
- 因此 funding 单独作为低频合约结构因子处理。

### 4.2 持仓变化：`oi_change_5`、`oi_change_15`、`oi_change_30`

计算方式：

```text
oi_change_N = log(open_interest_t / open_interest_{t-N})
```

含义：

- OI 上升：有新仓进入。
- OI 下降：有平仓或去杠杆。

用途：

- 价格上涨 + OI 上升，可能表示多头主动加仓。
- 价格上涨 + OI 下降，可能只是空头平仓。
- 价格下跌 + OI 上升，可能表示空头主动加仓。

这里使用 `5/15/30`，因为 OI change 可以自然按过去 N 分钟变化来解释。

### 4.3 合约基差：`basis_z_5`、`basis_z_15`、`basis_z_30`

先计算原始 basis：

```text
basis_t = swap_close_t / spot_close_t - 1
```

再按窗口做相对异常：

```text
basis_z_N = (basis_t - rolling_mean(basis, N)) / rolling_std(basis, N)
```

含义：

- basis 为正：永续合约相对现货溢价。
- basis 为负：永续合约相对现货折价。
- `basis_z_N` 表示当前 basis 相对过去 N 分钟是否异常。

用途：

- 捕捉永续合约相对现货的拥挤程度。
- 有助于判断合约市场是否过热或过冷。

## 5. 暂不使用的因子

第一版不使用以下因子：

| 因子 | 暂不使用原因 |
| --- | --- |
| `amplitude` | 属于 K 线形态，用户明确不想使用 |
| `upper_shadow` | 属于 K 线形态 |
| `lower_shadow` | 属于 K 线形态 |
| `body_ratio` | 属于 K 线形态 |
| `rsi_14` | 窗口不符合 `5/15/30` |
| `atr_14` | 窗口不符合 `5/15/30` |
| `ma60_dev` | 窗口不符合 `5/15/30`，且对 h30 可能偏慢 |
| 传统 `macd_12_26_9` | 窗口不符合 `5/15/30` |
| `roc_*` | 与 `ret_*` 信息高度重复 |
| `obv_z` / `mfi_*` | 量价指标解释成本较高，先放到 v2 消融 |
| `vwap_dev` | 不同数据源的 amount/volume 口径可能不一致，先放到 v2 消融 |
| `amount_z` | 成交额重要，但第一版先控制变量；可作为 v2 加入 |

## 6. 尺度处理建议

本方案统一的是“指标窗口”，即所有技术指标只使用 `5/15/30`。数值尺度建议另做统一处理：

- 比率类因子保持无量纲，例如 `ret_N`、`maN_dev`、`boll_z_N`。
- RSI 统一缩放到 `[-0.5, 0.5]`。
- MACD 除以 `close_t`，避免价格量纲影响。
- OI change 使用 log ratio。
- basis 使用 z-score。
- 所有因子最后统一 `fillna(0)` 并 `clip(-5, 5)`。

如果后续要进一步统一数值分布，可以在打包阶段增加统一 rolling z-score 或全局标准化。但这属于“归一化策略”，不要和 `5/15/30` 的“指标窗口”混在一起。

## 7. 维度汇总

| 数据类型 | 因子数量 | 说明 |
| --- | ---: | --- |
| 现货 | 21 | 17 个技术指标 + 2 个市场环境 + 2 个时间特征 |
| 永续 | 28 | 现货 21 个 + 7 个合约结构因子 |
| 当前模型兼容 | 32 | 现货需要 11 个 pad，永续需要 4 个 pad |

如果后续决定真正减少模型输入维度，可以把 `n_exog` 从 32 改小。但第一阶段更稳妥的做法是：保持 `n_exog=32`，只改变前面有效因子的语义，剩余列填 0。
